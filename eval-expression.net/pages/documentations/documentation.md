# Documentation

## Methods

### Main Methods

| Category          | Name            | Description                                                                                                                                                                                                                                                                         |
| ----------------- | --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Instance Methods  | context.Execute | The context.Execute method lets you execute an expression at runtime and return the result. You can use the method with a generic type to return a strongly typed value.                                                                                                            |
| Instance Methods  | context.Compile | The context.Compile method lets you compile an expression at runtime and return a delegate. You can use the method with a Func or Action to return a strongly typed delegate.                                                                                                       |
| Static Methods    | Eval.Execute    | The Eval.Execute static method lets you execute an expression at runtime. Under the hood, this is equivalent to calling the EvalManager.DefaultContext.Execute method (See context.Execute).                                                                                        |
| Static Methods    | Eval.Compile    | The Eval.Compile static method lets you compile an expression at runtime. Under the hood, this is equivalent to calling the EvalManager.DefaultContext.Compile method (See context.Compile).                                                                                        |
| Extension Methods | string.Execute  | The string.Execute extension method lets you execute a string expression at runtime. For example, "1+2".Execute() will return 3. Under the hood, this is equivalent to calling the EvalManager.DefaultContext.Execute method (See context.Execute).                                 |
| Extension Methods | string.Compile  | The string.Compile extension method lets you compile a string expression at runtime. For example, "1+2".Compile() will return a Func<object> that will return 3. Under the hood, this is equivalent to calling the EvalManager.DefaultContext.Compile method (See context.Compile). |
| Helper Methods    | AddMethod       | The AddMethod method allows you to create a method or extension method dynamically. You can later use added methods when evaluating or compiling an expression.                                                                                                                     |
| Helper Methods    | Tokenize        | The Tokenize method parses an expression and returns a list of TokenInfo. The method can be helpful to understand better how the library parses an expression or to extract some specific part, such as finding all string tokens.                                                  |

## Eval Manager

| Category    | Name           | Description                                                                                                                                                                                                                                                                                                                                 |
| ----------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| EvalManager | Cache          | The Cache option lets you get or set the memory cache where the delegate of a compiled expression is cached. The Cache default value is a new MemoryCache.                                                                                                                                                                                  |
| EvalManager | DefaultContext | The DefaultContext option lets you get or set the default context used under the hood by methods such as Eval.Execute, Eval.Compile, string.Execute, string.Compile and LINQ Dynamic. Generally, you use the DefaultContext to set options or register type to the context. For example, EvalManager.DefaultContext.IsCaseSensitive = true. |
